# 学习

  • 一直在说SPA 单页面应用，什么是单页面应用？

    • 单页面应用程序将所有的功能局限于一个web 页面中，仅在该web 页面初始化时加载相应的资源（HTML、 JavaScript 和CSS）。
    • 一旦页面加载完成了，SPA 不会因为用户的操作而进行页面的重新加载或跳转。而是利用JavaScript 动态地变换HTML 的内容，从而实现页面与用户的交互。
    • SPA 单页面应用程序的优缺点
    • 3个优点
      • 良好的交互体验：
        • 单页应用的内容的改变不需要重新加载整个页面
        • 获取数据也是通过Ajax 异步获取
        • 没有页面之间的跳转，不会出现“白屏现象”
      • 良好的前后端工作分离模式
      • 减轻服务器的压力，服务器只提供数据，不负责页面的合成与逻辑的处理
    • 2个缺点
      • 首屏加载慢
        • 解决思路
          • 路由懒加载
          • 代码压缩
          • CDN 加速
          • 网络传输压缩
      • 不利于SEO
        • SSR 服务器端渲染

  • 整理了下z-index的问题
    • z-index: 0 和z-index: auto的区别？
      • 官方原话：因为z-index:auto和z-index：0对于堆栈上下文排序，它们被认为是相等的，它们可以按DOM顺序交错。 需注意：z-index:0可以创建层叠上下文，也就是说会影响内部子元素与z-index:0同级元素z-index:auto内子元素的权重。
      • 什么意思呢？就是指z-index: 0会创建一个层级，里面的元素是显示在z-index:0的层级上，而z-index:auto则不会，里面的元素是直接显示在DOM文本流这个层级上
