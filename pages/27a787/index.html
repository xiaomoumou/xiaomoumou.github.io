          yarn和npm的常用指令

          整理下yarn和npm的常用指令，方便自己查阅，直接看指令

          # 历史

          • npm

            • npm是围绕着 语义版本控制（semver）的思想而设计。
            • 主版本号.次版本号.补丁版本号
              • 主版本号： 当API发生改变，并与之前的版本不兼容的时候
              • 次版本号： 当增加了功能，但是向后兼容的时候
              • 补丁版本号：当做了向后兼容的缺陷修复的时候

          • Yarn

            • Yarn 是为了弥补 npm 的一些缺陷而出现的。

          # yarn和npm的差异

          • 定义
            • npm的缺点
              • 速度慢:删除node_modules，install安装依赖的时候都很慢。

              注意

              问：为什么慢？ 答：npm 2会安装每一个包所依赖的所有依赖项。如果我们有这么一个项目，它依赖项目A，项目A依赖项目B，项目B依赖项目C，那么依赖树将如下所示：

              node_modules
- package-A
-- node_modules
--- package-B
----- node_modules
------ package-C
-------- some-really-really-really-long-file-name-in-package-c.js
              1
              2
              3
              4
              5
              6
              7

              npm 3采用了扁平依赖关系树来解决这个问题，所以我们的3个项目结构现在看起来如下所示： bash node_modules - package-A - package-B - package-C -- some-file-name-in-package-c.js 一般采用淘宝镜像可以加快速度

              • 同一个项目，安装的时候无法保持一致性。由于package.json文件中版本号的特点，下面三个版本号在安装的时候代表不同的含义。
                "5.0.3",  // 表示安装指定的5.0.3版本
"~5.0.3", // 表示安装5.0.X中最新的版本
"^5.0.3"  // 表示安装5.X.X中最新的版本。
                1
                2
                3
            • yarn的优点
              • 速度快：npm是按队列下载，yarn是并行执行
              • 离线模式：如果之前已经安装过一个软件包，用Yarn再次安装时之间从缓存中获取，就不用像npm那样再从网络下载了。
              • 版本统一：yarn安装依赖时会产生yarn.lock文件，锁定依赖版本

          # 指令

          含义 npm yarn
          安装所有包 npm install yarn add vue
          安装指定包 npm install vue --save yarn add vue
          卸载指定包 npm uninstall vue --save yarn remove vue
          卸载指定包 npm update --save yarn upgrade

          -g 是全局安装

          –save，或-S，作用于生产环境， 写入dependencies

          –save-dev，或-D，作用于开发环境，写入devDependencies

          项目打包时只会打包生产环境的依赖
