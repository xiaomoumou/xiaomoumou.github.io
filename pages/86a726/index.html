  • 1、进入消息页面，显示当前用户的会话列表

  • 2、进入聊天页面，发送接收消息

# 1、进行消息发送接收的技术 socket

# socket的基础概念

http协议只管发送消息，发送完就结束了。而socket协议是长连接，一直保持连接状态。

  • 创建 使用new websocket()的方式新建一个socket实例
  • 发送消息
  • 接收消息
  • 保持连接
    • 心跳机制
  • 关闭

# 发送接收流程

a发消息给b时会发生什么

a——>服务器——>b

a通过socket实例的send()方法发送，。

b通过socket的事件监听onmessage获取到信息。

# 建立socket连接的验证

正式使用时，一般会先发送一个消息给服务器，带上本次访问的信息。

服务器根据信息进行验证，判断是否是正常使用，成功的话会建立连接，然后返回信息给使用者。

a——>服务器 (请求建立socket连接)

服务器——>a (建立socket连接成功/失败)

失败的话，考虑是否需要重连

# 心跳机制

首先明确上面的发送接收机制。

为了检测客户端和服务端是否处于正常的链接状态，每隔一段时间发一条固定格式的信息，同时客户端会确认服务器端是否还活着，如果还活着的话，就会回传一个数据包给客户端来确定服务器端也还活着。

这个就是心跳机制。

# 项目里的一些问题

  • 什么时候建立socket连接？
    • 用户登陆完之后建立 socket连接，确保在所有页面都能收到消息
  • 接受消息之后怎么传递给所有页面？
    • 目前我是通过vuex定义了unreadItem存储所有未读消息

# 会话列表

  • 会话列表界面
    • 接口返回上线时的数据，所需内容：
      • 未读消息数，
      • 最后一条消息内容，
      • 最后一条消息时间
    • 过滤数据
      • 接受到消息时过滤一下
      • 和本地缓存的消息过滤一下
    • socket 接受到新的消息，可能是一个新的会话，需要重新获取会话列表

# 聊天框

# 发送功能

  • 文字，直接提交字符串
  • 表情，需要处理，通过正则匹配判断是否为表情
    • 提交时，可以把类型设为 0，为了方便 文字 + 表情 一起发送
    • 通过表情索引，远程请求 gif 文件
  • 图片，消息类型为 1，需处理
    • input-file 选择本地图片，上传至服务器，返回图片在服务器的 url
      • 第三方文件服务器，如 oss
      • 公司内部文件服务器
    • 本地预览
      • 通过请求远程服务器获取远程 url
      • 通过转码成 base64 直接显示
    • 输入框怎么显示图片？
      • 发送表情时，输入框只有一些文字。不支持图片显示？

# 聊天内容显示

  • 发送消息时，添加至当前数据，本地缓存
  • 接受消息时，监听 socket，添加至当前数据，本地缓存
  • 加载消息
    • 进入页面时，获取第一页，滚动至底部
    • 分页显示，一次 10 条，向上滚动加载下一页，数据加载至头部
    • 加载下一页之后，滚动到上一页的最后一条消息
  • ui行高问题
    • 消息里，表情和文字高度不一致，会影响行高，qq 也没有解决这个问题
