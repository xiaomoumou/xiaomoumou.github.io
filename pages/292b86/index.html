# (一)命名规范

  • 项目命名、目录命名、JS、CSS、SCSS、HTML、PNG 文件命名

统一使用 kebab-case 命名（即短横线命名）

【特殊情况】：VUE项目中的components组件目录，使用大驼峰命名

  • 命名严谨性 禁止在命名时使用拼音、中文

# 1.2.1 HTML 类型

# 1.2.2 缩进

缩进使用 2 个空格（一个 tab）

嵌套的节点应该缩进。

# 1.2.3 分块注释

在每一个块状元素，列表元素和表格元素后，加上一对HTML注释。

# 1.2.4 语义化标签

优先使用语义化标签，避免一个页面都是div或者p标签。

<!-- 【推荐】 -->
<header></header>
<footer></footer>

<!-- 【不推荐】 -->
<div></div>
<div></div>

# 1.2.5 引号

# (三) CSS 规范

# 1.3.1 命名

类名使用小写字母，以中划线分隔

id 采用驼峰式命名

scss 中的变量、函数、混合、placeholder 采用驼峰式命名

【注意】：ID 和 class 的名称总是使用可以反应元素目的和用途的名称，或其他通用的名称

# 1.3.2 选择器

不推荐使用后代选择器，推荐使用子选择器

/* 【推荐】 */
.content > .title {
  font-size: 2rem;
}

/* 【不推荐】 */
.content .title {
  font-size: 2rem;
}
