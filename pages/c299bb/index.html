# 使用流程

# 使用eslint依赖

  • 1、先使用npm init -y初始化项目，生成package.json文件。

  • 2、新建index.js文件，写一些代码。

  • 3、使用npx eslint --init(需要提前安装全局eslint，使用npm i eslint -g),会自动生成.prettierrc文件，该文件包含一些基本配置，并且会在package.json安装相应依赖。

  • 4、使用eslint index.js格式化代码，终端会报错，eslint只坚持语法问题，格式方面默认不关注，需要手动配置

    • 4.1 如果需要修改格式问题，可以去配置.eslintrc.js文件
    //.eslintrc.js
// 0是忽略，1是警告，2是报错
  "rules": {
      "quotes": 2,
      "semi": 1,
      "no-console": 1,
      "space-before-function-paren": 0
  }

# vscode中的eslint插件

  • 1、vscode中的eslint插件可以实时查看代码质量。下载该插件查看效果

    此时会出现波浪线，红色代表错误，黄色代表警告

# prettier

  • 1、安装prettiernpm i prettier -D，然后执行npx prettier --write index.js

可以格式化index.js页面的代码样式

会出现格式化两次的问题，一次是按照eslint的格式，一次是prettier的格式，后者会覆盖前者。

此外，eslint只关心javascript,prettier可以格式化html、css、js、vue等

  • 2、自定义prettier格式化规则，创建.prettierrc.js文件（也可以是.prettierrc，类型是json），进行配置
//.prettierrc.js文件

module.exports = {
    semi: false,
    singleQuote: true,
}

# 使用ESLint + Prettier的流程

安装

npm i prettier eslint-plugin-prettier -D
1

eslint-plugin-prettier插件会调用prettier对你的代码风格进行检查，其原理是先使用prettier对你的代码进行格式化，然后与格式化之前的代码进行对比，如果过出现了不一致，这个地方就会被prettier进行标记。

//.eslintrc.js
{
  "plugins": ["prettier"],
  "rules": {
    "prettier/prettier": "error" // 被prettier标记的地方抛出错误信息
  }
}

修改webpack的配置，就能在启动webpack-dev-server的时候，每次保存代码同时自动对代码进行格式化。

// webpack.config.js
const path = require('path')
module.exports = {
  module: {
    rules: [
      {
        test: /\.(js|vue)$/,
        loader: 'eslint-loader',
        enforce: 'pre',
        include: [path.join(__dirname, 'src')],
        options: {
          fix: true
        }
      }
    ]
}
