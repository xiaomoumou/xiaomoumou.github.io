给自己的一些规范

深感每次写页面想法不同，规范也乱七八糟，所以整理一下约束自己的规范 平时主要是通过element-ui进行开发

  • 尺寸统一设置为mini 优先从cookies中获取
// set element-ui default size
Vue.use(ElementUI, {
  size: Cookies.get('size') || 'mini' 
})

# 事件调用

在<el-dialog>组件内使用<el-form>时，如果需要清理表单，统一在close钩子事件中调用resetField() 方法

提示

1、在<el-dialog>组件内使用<el-form>时，第一时间使用resetField()会报错的问题。

<el-dialog>组件第一次打开前，slot部分不会被渲染到DOM,所以要操作DOM的话，需要在open事件回调中进行，清理表单，统一在close事件中进行

2、表单赋值之后，如果每次使用resetField()都会变成第一次赋值的值，可能是因为没有正确赋值。

this.dialogVisable = true
this.form.name='二'
this.form.age=25
1
2
3

这种情况下dom首次渲染时，初始值会被记录成编辑时加载的数据。如果要正确赋值，需要使用$nextTick()在dom渲染之后赋值

this.dialogVisable = true
this.$nextTick(()=>{
  this.form.name='二'
  this.form.age=25
})

# 按钮

  • 封装一些以<el-button>为基础的公共组件，方便后续统一调整某一种类型的按钮，比如添加类的按钮统一修改为蓝色，圆角样式
    • 返回按钮 primary
    • 筛选按钮 primary
    • 添加按钮 success
    • 删除按钮 danger
    • 编辑按钮 primary
  • 不可编辑的，使用透明背景，边框线条

# 分页

  • 封装分页组件<el-pagination>
    • 参考element-admin,把所有的分页相关的data、methods放置到一个公共的vue文件内部
  • 当某页只含有一页时，不显示分页功能
  • 超出一行时，才会显示分页栏

# 子页面

  • 绝大多数需要考虑添加返回按钮
  • 考虑是否需要使用<keep-alive>

# 图片

  • 考虑图片比例，比例需要固定，避免扭曲变形
  • 部分需要放大的图片，考虑全屏功能或者局部放大
  • 封装成指令，方便随时可以在全局调用

# 数字

# 小数点的问题

处理需要统一计算的小数时,比如

0.55转换为55%，正常考虑是0.55*100=55%

实际上，在js里 0.55*100=555.00000000000001

所以如果需要统一计算小数，使用toFixed()函数，封装全局过滤器

let ratioFilter = (input,num)=>{
  if (input === null|| input===undefined) {
    return '--'
  }
  return Number(input*100).toFixed(num)+'%'
}
