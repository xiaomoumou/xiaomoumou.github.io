上传文件分为哪些操作？

  • 本地操作

# 一、本地操作

# 1.【获取本地文件】

input 标签之 file 类型

# 1.1 基础用法

input 标签 type 为 file 时，input 表现性状为上传文件样式。

<!-- html -->
<input type="file" multiple id="imgLocal" accept="image/*" />
<script>
// js
  let inp = document.querySelector('#imgLocal')
  inp.onchange = function(e) {
    // 1、可以通过访问 dom 上的属性 files 来访问
    let fileList = document.querySelector('#imgLocal').files
    console.log(fileList) 
    // 2、可以通过访问e.target.files来访问
    console.log(e.target.files)
  }
</script>

关于input[type:file] 的属性

  • accept：指定选择文件类型的范围。默认为所有文件类型
  • capture：当文件类型为图片或视频且在移动端时，此属性才有意义。
    • capture = ‘user’ 调用前置摄像头
    • capture = ‘environment’ 调用后置摄像头
  • multiple：一个 Boolean 值，如果存在，则表示用户可以选择多个文件
  • files：返回一个 FileList，列出每个所选文件对象。除非 multiple 指定了属性，否则此列表只有一个成员。主要用于 JS 操作。

# 2.【读取文件】

FileReader 对象

# 2.1 基础用法

FileReader 对象允许 Web 应用程序异步读取存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用 File 或Blob 对象指定要读取的文件或数据。

FileReader 对象常用事件和方法

  • FileReader.onload
    • 处理 load 事件。该事件在读取操作完成时触发
  • FileReader.readAsDataURL()
    • 开始读取指定的 Blob 中的内容。一旦完成，result 属性中将包含一个 data: URL 格式的字符串以表示所读取文件的内容。
  ...
  const fileReader = new FileReader()
  fileReader.readAsDataURL( file )

# 2.2 显示读取本地文件的进度

FileReader 有另一个事件，称为进度(progress)，知道已加载了多少。

巨坑，通常表现为只监听一次。 实际上每过50ms左右，就会触发一次progress事件，对于较大的文件可以利用progress实现进度条 w3c.github.io (opens new window)

  // 监听大文件上传进度
  fileReader.addEventListener( 'progress', ( event ) => {
    if ( event.loaded && event.total ) {
      // 计算完成百分比
      const percent = ( event.loaded / event.total ) * 100;
    }
  } );

# 2.3 创建文件URL的方式

之前提到，通过FileReader 对象的result，得到文件的base64编码，可以当做url。

有一个称为 URL.createObjectURL() 的特殊方法，可以从文件创建唯一的URL，您也可以使用 URL.revokeObjectURL() 方法释放它。

  let url = ''
  fileReader.addEventListener( 'loadend', function () {
    // 返回base64格式的图片url
    let baseUrl = fileReader.result
    // 返回blob对象的图片url
    let blobUrl = window.URL.createObjectURL(file)
    url = baseUrl
    console.log(baseUrl)
    url = blobUrl
    console.log(blobUrl)
  } )

关于baseUrl和blobUrl的区别，如图所示

<!-- html -->
<input type="file" multiple id="imgLocal" accept="image/*" />
<script>
  let inp = document.querySelector( '#imgLocal' )
  inp.onchange = function ( e ) {
    // 1、可以通过访问 dom 上的属性 files 来访问
    let fileList = document.querySelector( '#imgLocal' ).files
    console.log( fileList )
    // 2、可以通过访问e.target.files来访问
    console.log( e.target.files )
    const files = Array.from(e.target.files);

    // dom对象不可直接使用forEach()
    files.forEach( file => {
      const { size, name, type } = file;
      // 检查文件大小是否大于1MB。
      if ( size > 1024 * 1024 ) {
        console.log( '文件大于1MB' )
      }
      // 检查上传的文件是否为图片类型
      const imgTypes = ['image/png', 'image/jpg','image/jpeg']
      if ( !imgTypes.includes( type ) ) {
        console.log(type)
        console.log( '文件非图片类型' )
      }
      const fileReader = new FileReader()
      fileReader.readAsDataURL( file )
      // 监听大文件上传进度，50ms触发一次
      fileReader.addEventListener( 'progress', ( event ) => {
        if ( event.loaded && event.total ) {
          // 计算完成百分比
          const percent = ( event.loaded / event.total ) * 100;
        }
      } );
      let url = ''
      fileReader.addEventListener( 'loadend', function () {
        // 返回base64格式的图片url
        let baseUrl = fileReader.result
        // 返回blob对象的图片url
        let blobUrl = window.URL.createObjectURL(file)
        url = baseUrl
        console.log(baseUrl)
        url = blobUrl
        console.log(blobUrl)
      } )
    } )
  }
</script>

# 1.【组装表单对象】

FormData 对象 FormData 对象的作用

  • 1.用一些键值对来模拟一系列表单控件：即把 form 中所有表单元素的 name 与 value 组装成一个 queryString
  • 2.异步上传二进制文件。
let formData = new FormData()
formData.append(name, value, filename)

关于 FormData()实例的参数

  • name:属性名
  • value:属性值，在我们这里则指 file 数据。
  • filename:当第二个参数为 file 或 blob 时，告诉服务器的文件名。Blob 对象的默认文件名是“blob”。

# 使用form表单上传

要添加属性 enctype="multipart/form-data"

form表单的enctype属性（method="post" 时才使用 enctype 属性）

  • 作用：告知服务器请求正文的MIME类型，规定在发送到服务器之前应该如何对表单数据进行编码。（请求消息头：Content-Type作用是一致的）
  • 可选值：
    • application/x-www-form-urlencoded(默认)：发送前对所有字符进行编码
    • multipart/form-data: 使用有文件上传控件的表单时，该值是必需的。
    • text/plain：将空格转换为 "+" 符号，但不编码特殊字符
