  • 在使用eslint的时候，使用的airbnb-base标准，遇到一个换行符问题
Expected linebreaks to be 'LF' but found 'CRLF'
1

大致意思是说“由于不同系统不同工具下，换行符不同”

在各操作系统下，文本文件所使用的换行符是不一样的。 UNIX/Linux 使用的是 0x0A（LF），早期的 Mac OS 使用的是0x0D（CR），后来的 OS X 在更换内核后与 UNIX 保持一致了。 但 DOS/Windows 一直使用 0x0D0A（CRLF）作为换行符。 解决的方案

// .eslintrc.js
{
  "rules":{
    //允许windows开发环境
    "linebreak-style": [0 ,"error", "windows"]
    // "linebreak-style": ["off", "windows"]
  }
}

# 2022-01-12

# 学习

  • 关于在webapck项目中css使用别名出错的问题
    //js中:
  import tool from '@/utils/xxx' //(可以引用)
  
//css中:
  @import '@/assets/css/reset.sass' //(报错)
    1
    2
    3
    4
    5

    • 【原因】：css 文件会被用 css-loader 处理，这里 @import 后的字符串会被 css-loader 视为绝对路径解析，因为我们并没有添加 css-loader 的 alias，所以会报找不到 @ 目录。

    • 【解决】：

      • 为 css-loader 添加 ailas 的路径
      • 引用路径的字符串最前面添加上 ~ 符号
        • 如 @import "~@/style/theme"
        • Webpack 会将以 ~ 符号作为前缀的路径视作依赖模块而去解析，这样@ 的 alias 配置就能生效了。

  • Object.create()

    • 参考：Object.create() (opens new window)
    • 语法：Object.create(proto, [propertiesObject])
      • proto : 必须。表示新建对象的原型对象
      • propertiesObject : 可选。 添加到新创建对象的可枚举属性
      // new Object() 方式创建
  var a = {  rep : 'apple' }
  var b = new Object(a)
  console.log(b) // {rep: "apple"}
  console.log(b.__proto__) // {}
  console.log(b.rep) // {rep: "apple"}

  // Object.create() 方式创建
  var a = { rep: 'apple' }
  var b = Object.create(a)
  console.log(b)  // {}
  console.log(b.__proto__) // {rep: "apple"}
  console.log(b.rep) // {rep: "apple"}

# 学习

  • 发现线上使用input-file时存在一个问题，提示 files.forEach is not a function

    • files是input-type文件的上传部分， const files = e.target.files
    • 错误原因：
    • xxx.forEach is not a function（DOM集合--类数组对象转化为数组） (opens new window)
      • Array.from(elems)
      • Array.prototype.slice.call(elems)
      • [ ...elems ]
      • Array.prototype.forEach.call(elem,callback)
        • call/bind/apply
    • 总结: input-file在原生js和vue的不同表现,需要把File对象转换为数组对象
